ðŸ‘‰ `.gitignore` me jo likha hota hai, Git usko track nahi karta

### "/" Route ka Concept
- Agar route me sirf `/` likha hai:
  - Toh uske baad URL me **kuch bhi likho**, wahi route hit hoga
- Ye baaki saare routes ko **override** kar sakta hai

ðŸ‘‰ Isliye `/` route hamesha **last me** likhna chahiye

---

### Browser aur API Calls
- Jab hum browser me URL type karte hain:
  - Browser **GET API call** karta hai
- Browser se **POST request directly possible nahi hoti**

ðŸ‘‰ Browser API testing ke liye worst option hai

ðŸ‘‰ Best practice:

# ðŸ”¹ Advanced Routing Concepts (Express.js)

### Route Patterns ka Matlab (Hinglish)

#### `"ab?c"`
- `b` optional hai  
- Matches:
  - `ac`
  - `abc`

---

#### `"ab*cd"`
- `*` ka matlab kuch bhi ho sakta hai  
- Matches:
  - `abcd`
  - `abXYZcd`

---

#### `"/a/"`
- Sirf `/a` route ko match karega  

---

#### `"/.*fly3/"`
- Regex based route  
- Koi bhi route jo `fly3` pe end ho  

---

#### `"a(bc)?"`
- `bc` optional hai  
- Matches:
  - `a`
  - `abc`

---

#### `"a(bc)+d"`
- `bc` kam se kam ek baar hona chahiye  
- Matches:
  - `abcd`
  - `abcbcd`

---

#### `"/user"`
- Simple static route  
- Sirf `/user` URL pe hit hoga  

---

#### `"/user/:userid"`
- Dynamic route  
- `userid` ek variable hai  

Example:
- `/user/101`
- `/user/ayush`

ðŸ‘‰ `userid` ko hum request params se access kar sakte hain
