## Why Password Encryption is Mandatory

Password kabhi bhi:
- Plain text mein database mein store nahi hona chahiye

Agar database leak ho jaye:
- Plain password sabko mil jaayega
- Users ke bank, email, social accounts hack ho sakte hain

Isliye:
> Password hamesha **hashed form** mein store hona chahiye

## Rounds (Cost Factor)

Rounds decide karte hain:
- Hash kitna slow hoga
- Kitni baar internal hashing repeat hogi

More rounds:
- Zyada security
- Thoda slow

Production apps usually 10â€“12 rounds use karte hain.

## How bcrypt Works Internally

1. Plain password liya
2. Random salt generate hua
3. Password + salt combine hua
4. Multiple rounds of hashing hue
5. Final encrypted hash database mein store hua

## Why Validation Before Hashing

Password hashing se pehle:
- Check karo password empty na ho
- Length strong ho
- Format valid ho

Invalid password ko hash karna waste hota hai.

---

## Signup Flow (Secure Version)

1. Client password bhejta hai (plain)
2. Backend validate karta hai
3. bcrypt se password hash hota hai
4. Hash database mein save hota hai
5. Plain password discard ho jaata hai

---

## Login Flow (Secure Version)

1. User email + password bhejta hai
2. Database se user ka hash nikala jata hai
3. bcrypt compare karta hai:
   plain password vs stored hash
4. Agar match â†’ login success
5. Agar mismatch â†’ login fail

Plain password kabhi bhi database se compare nahi hota.

---

## Why We Never Decrypt Passwords

Hashed passwords:
- Decrypt nahi hote
- Sirf compare hote hain

Security ka golden rule:
> Password ko decrypt nahi, compare karo
